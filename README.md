## О проекте
IYHAN SHOP – это SPA интернет-магазин танков. Проект построен на Node.js + Express (бэкенд) и TypeScript (клиент). Проект предполагает авторизацию учётных записей для пользователей, обширный каталог танков, корзину выбранных позиций к покупке и её оплату с информацией о доставке.

## Добавление нового функционала
Для добавления своей части проекта необходимо выбрать свою ветку, после чего подхватить базовую структуру из ветки main.
1. Клонировать репозиторий: git clone https://github.com/pltdeveloper393/L_Shop.git
2. Перейти в ветку main и склонировать содержимое к себе в ветку
3. Разрабатывать свою часть проекта на основе базовой структуры

В базовую структуру каждой ветки входят:
- Корневые файлы:
	- package.json                    # зависимости проекта (!)
	- tsconfig.json                   # настройки TypeScript для сервера (!)
	- tsconfig.client.json            # настройки TypeScript для клиента (!)
	- .gitignore                      # игнорируемые файлы (!)
	- README.md 					  # документация (!)

- Сервер (server):
	- controllers 					  # папка с логикой обработки запросов
	- middleware 					  # папка с промежуточными обработчиками
	- models 						  # папка с моделями данных
	- routes 						  # папка с маршрутами
	- index.ts 						  # главный файл запуска сервера (создавать свой)

- Клиент (src):
	- pages 						  # папка со страницами приложения
	- services 						  # папка с сервисами (с api файлами)
	- types 						  # папка с TypeScript типами
	- main.ts 						  # точка входа клиента (запускает приложение)
	- router.ts 					  # главный роутер
	
- Статика (public):
	- js 							  # скомпилированные файлы (создаётся автоматически)
	- images						  # папка хранения изображений (если нужны)
	- index.html 					  # базовый HTML (куда помещается SPA-страницы)
	- styles.css 					  # общий css-файл проета

Каждый участник репозитория при разработке своей части должен:
- создать аналогичную структуру своей части проекта в своей ветке

- разработать серверную часть (server):
	- добавить файлы контроллеров, моделей, маршрутов и middleware в соответствующие папки
	- создать собственный файл запуска в корень папки server, аналогичный файлу server/index.ts

- разработать клиентскую часть (src):
	- добавить страницы в src/pages
	- создать собственный файл API в папке src/services , аналогичный файлу api.ts
	- создать собственный файл типов в папке src/types , аналогичный файлу auth.ts
	- создать собственный файл маршрутов в корень папки src, аналогичный файлу src/router.ts

- разработать статику (public):
	- подключить доп стили в public/index.html и поместить в корень папки public свой css-файл для вашей страницы (если такой имеется)



## Пример внедрения новой страницы (страница профиля пользователя)

### Шаг 1: Создание файла страницы
`src/pages/ProfilePage.ts` - тут HTML и логика страницы для SPA

### Шаг 2: Добавление маршрута в роутер
В `src` создать файл `router.profile.ts` и добавить импорт:
```typescript
import { renderProfilePage } from './pages/ProfilePage.js';
router.addRoute('/profile', renderProfilePage);
```
### Шаг 3: Добавление функций в API проекта (если требуется)
В `src/services` создать файл `api.profile.ts` и добавить:

```typescript
getProfile: () => request('/auth/profile')
```

### Шаг 4: Добавить кнопку перехода
В нужной странице (например, MainPage.ts):

```html
<button onclick="router.navigateTo('/profile')">Профиль</button>
```

### Шаг 5: Создать css-файл под вашу страницу
public/profile-page.css и подключите в index.html

## Как проверить, что всё работает

1. Запустите сервер: `npm run dev`
2. Откройте `http://localhost:3000/путь`
3. Откройте консоль браузера (F12) - нет ошибок
4. Проверьте Network вкладку - запросы уходят
